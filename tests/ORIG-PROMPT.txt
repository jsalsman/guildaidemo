Build a single-page, microphone-enabled EFL pronunciation evaluation web application focused on syllable stress for noun/verb stress-shift pairs, AND expose an A2A-compatible “remote agent” interface (discovery + JSON-RPC) that lets other agents call this service programmatically.

The app must:

- Let a learner choose 1 of 5 paragraphs stored in `5-paragraph-syllable-stress-test_NV.txt`.
- Paragraphs contain target words annotated with `[N]` or `[V]` (only two-syllable words are annotated).
- Record the learner reading the selected paragraph using WebRTC microphone input.
- Record WAV in-browser at exactly: 16,000 Hz sample rate, 16-bit PCM, mono; produce a valid `audio/wav` file containing linear PCM (L16).
- Upload the WAV bytes to a Python Flask backend for analysis.
- Use Deepgram STT to obtain word endpoints and confidence scores using: model `nova-2`, language `en-US`, punctuation disabled, diarization disabled.
- Tolerate deviations: do NOT assume the spoken output matches the paragraph text. Implement robust “inexact matching” between Deepgram words and the paragraph tokens; sometimes the match will be poor.

The primary deliverable is an MVP with a strong, clear results UI. No accounts, no persistence, no storage. One web page only. Web app and A2A integration only.

You MUST incorporate and adapt the reference logic from `score_server.py` in a meaningful way:
- Reuse the Deepgram transcription call structure and parsing style that produces a flat list of word timing objects `{word, start, end, confidence}`.
- Reuse the per-word audio segmentation approach: trim each word’s audio from the recording using Deepgram timings, add small padding, ensure 16kHz mono, and pass segments to the aligner.
- Reuse the PocketSphinx forced-alignment approach that yields phoneme segments with timestamps per phoneme.
- Reuse the pronunciation/phoneme matching approach (best pronunciation selection via phoneme match score) as the basis for identifying syllable core vowels.
- REQUIRED VISUAL FEATURE: compute `confidence_cubed = confidence ** 3` for each Deepgram word and use it to drive word BACKGROUND coloring in the UI. Confidence must ONLY be represented as background coloring (no waveforms, no timing bars).

Core algorithm requirements

A) Paragraph parsing and targets
- Parse `5-paragraph-syllable-stress-test_NV.txt` into 5 paragraph items.
- Each paragraph has tokens that may include markers like `record[N]` or `record[V]`.
- Create:
  1) a “display paragraph” with markers removed (for the user to read),
  2) a structured target list: `[{token_index, word, label}]` where label is N or V.
- Only the annotated words need stress evaluation; treat all annotated words as two-syllable.

B) Deepgram transcription (robust/inexact matching)
- Call Deepgram with:
  - model: `nova-2`
  - language: `en-US`
  - punctuation: disabled
  - diarization: disabled
- Parse to a flat array `deepgram_words` with:
  `{idx, word, start, end, confidence, confidence_cubed}`
- Normalize words for matching by: lowercase + strip punctuation/apostrophes + collapse whitespace.

C) Inexact alignment of Deepgram words to paragraph tokens
Because the learner may deviate from the prompt, implement robust mapping:
- Tokenize the display paragraph into normalized tokens.
- Tokenize Deepgram output words into normalized tokens.
- Use a simple sequence alignment / dynamic programming approach (edit distance / Needleman–Wunsch style) to align the two token sequences with insertions/deletions/substitutions.
- Produce a mapping from paragraph token indices -> deepgram word indices when possible.
- For each annotated target token, attempt to find a mapped Deepgram word index. If not found, mark as “missing” (no score, but report).

D) Stress scoring definition (voiced-core duration only)
Stress is determined by duration of each syllable’s VOICED CORE phoneme (vowel or diphthong), not total syllable duration.
- Provide an explicit list of CMUbet vowel/diphthong phonemes (AA, AE, AH, AO, AW, AY, EH, ER, EY, IH, IY, OW, OY, UH, UW) and handle stress-numbered variants (e.g., EH1, ER0).
For each annotated two-syllable word:
- Determine syllable1 core vowel phoneme and syllable2 core vowel phoneme.
- Compute each core duration from PocketSphinx phoneme alignment:
  `duration = phoneme_end - phoneme_start` for the core vowel phoneme in that syllable.
- Inferred stress:
  - if core1 > core2 -> stress 1
  - else -> stress 2
- Expected stress:
  - `[N]` => stress 1
  - `[V]` => stress 2
- If either core vowel phoneme cannot be found/aligned, mark “unaligned” and exclude from score denominator (but still show in UI).

E) Syllabification for targets (simple, deterministic, two-syllable only)
Make syllabification reliable for all annotated words:
- Include an in-repo mapping file `target_words_2syll_cores.json` containing entries for all annotated base words, each with:
  - one or more pronunciation variants (CMU phones list),
  - core vowel phoneme for syllable 1 and syllable 2.
- At runtime:
  - Use PocketSphinx aligned phoneme sequence for that word segment.
  - Choose the best matching variant by phoneme overlap score (as in the reference approach).
  - Locate the core vowels in the aligned phoneme list and extract their durations.

Backend requirements (Flask)

1) Routes
- GET `/`
  - Serve ONE HTML page with embedded JS/CSS (or static files) implementing the entire UI + developer docs section.
- GET `/api/paragraphs`
  - Returns JSON:
    - `request_id`
    - `paragraphs`: [{id, display_text, tokens, targets}]
- POST `/api/analyze`
  - Accept multipart form:
    - `paragraph_id`
    - `audio_wav` file upload (16kHz, 16-bit PCM, mono WAV)
  - Return JSON:
    - `request_id`
    - `deepgram_words`: array of `{idx, word, start, end, confidence, confidence_cubed}`
    - `alignment`: mapping between paragraph token indices and Deepgram word indices
    - `targets`: per-target results:
      {
        "word": "...",
        "label": "N"|"V",
        "expected_stress": 1|2,
        "inferred_stress": 1|2|null,
        "correct": true|false|null,
        "status": "ok"|"missing"|"unaligned",
        "core_durations": {"syll1": float|null, "syll2": float|null},
        "core_phones": {"syll1": "EH1"|null, "syll2": "ER0"|null},
        "deepgram_word_index": int|null,
        "deepgram_confidence": float|null,
        "deepgram_confidence_cubed": float|null,
        "feedback": "short actionable text"
      }
    - `score_summary`: {percent_correct, total_targets, scored_targets, missing_targets, unaligned_targets}
    - `render_words`: list of word spans for rendering the paragraph with background coloring:
      [{text, is_space, deepgram_idx, confidence_cubed, bg_norm, is_target, target_status, target_correct}]
- GET `/healthz`
  - Return JSON `{request_id, status:"ok"}` (request_id may be generated or passed in).
- GET `/.well-known/agent-card.json`
  - Return Agent Card JSON with request_id included as an extra field.
- POST `/a2a`
  - JSON-RPC 2.0 endpoint; include request_id in result or error payload.

2) Audio handling constraints
- Prefer in-memory processing:
  - Avoid writing temp files.
  - Use `io.BytesIO` for WAV bytes and in-memory slicing.
- If PocketSphinx forces file paths, use NamedTemporaryFile as last resort; always delete before responding.

3) Deepgram key and config
- Make sure the `DEEPGRAM_API_KEY` environment variable is set.
- Never commit secrets.
